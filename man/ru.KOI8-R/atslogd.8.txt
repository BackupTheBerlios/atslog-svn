NAME

ATSlog предоставляет удобный интерфейс с доступом через web-браузер для просмотра и анализа звонков различных моделей АТС.

SYNOPSIS

atslogd [-D dir] [-L logfile] [-s speed] [-c csize] [-p parity] [-f sbits]			[-t type] [-P PIDfile][-d] [-e] [-a] [-o] [-i address] [tcp:port] dev 

DESCRIPTION

atslogd - это модификация CDR Reader for PBXs v.1.09 (C) Alexey V. Kuznetsov, avk[at]gamma.ru, 2001-2002

для прораммы ATSlog версии @version@ build @buildnumber@ www.atslog.dp.ua

OPTIONS

-D dir		директория для текстовых лог-файлов; по умолчанию - текущая
-F filename     имя текстового лог-файла в который будут записаны данные
-L logfile      файл для сообщений об ошибках; по умолчанию stderr
-s speed        скорость передачи для последовательного порта; по умолчанию 9600
-c char_size    длинна символа (параметры последовательного порта); значения от 5 до 8
-p parity       чётность последовательного порта: e - even parity, o - odd parity, n - no parity.
-f stop_bits    number of stop bits; valid values 1 or 2                      
-d              выводить дополнительную отладочную информацию                              
-e              дублировать сообщения об ошибках в stderr (при наличии -L )
-a              write date at the beginning of file (for Definity type only)  
-o              выводить CDR данные дополнительно в stdout
-m              write log files on month-by-month instead of day-by-day basis 
-n              consider day in place of month and vice versa                 
-x number       максимальное колличество TCP соединений (по умолчанию: 1) смотрите /etc/hosts.allow, /etc/hosts.deny)
-w seconds      timeout before I/O port will be opened next time after EOF    
-i address      привязываться только на IP адрес (address) интерфейса (по умолчанию принимать на всех интерфейсах)
tcp:N       	где N номер TCP порта демона.
-b              демонизироваться.
-P              PID файл. /var/run/atslogd.pid по умолчанию.
				                                                                              
Ипользуйте стандартные настройки libwrap для контроля доступа TCP соединений. Подробнее в документации на /etc/hosts.allow и /etc/hosts.deny
